Input: Import from Database Tables

Define metrics

Different tables are used as input for python (and can be modified through the admin views):

Define metrics: in this table you can define all the metrics. This corresponds to the excel sheet “All_metrics” form the old Upload.xlsx-file.

If the metric is created before the multiscraper-file is uploaded, this metric will be computed.

For this table, version_control has also been added, so that any change can be reverted.

As input for the metric, you can use either datapoints (defined in Define datapoints) or outputs from the scrapers (defined in Scraper variables). A wrong definition of a scraper will not make the program crash, but the problem will be visible on the interface.

There are 5 formula-type at the moment:

  • None: the output will be equal to an empty string (‘’)
  • “=DP1/DP2”: simple quotient of two datapoints (or scraper outputs). Some checks will be performed, such as division by 0, replacing “nan”-values by 0, etc.
  • “=DP1”
  • “(DP1+DP2)/DP3”: same as DP1/DP2, but using a sum
  • “mult * DP1/DP2”: same as DP1/DP2, but using a multiplication factor

Arg1, arg2, arg3, arg4 are used for the formula-types.

Show_metrics, defines if this metric should be computed or not.
