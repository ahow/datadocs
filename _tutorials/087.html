Select which scrapers will run Using Scraper variables

To select which scrapers will run, there are different options:

The better method is to use Scraper variables directly

5 actions have been defined:

  • to check the checkboxes for all of the records,
  • to deselect them all,
  • to check the checkboxes for selected records and
  • to uncheck for the selected records
  • Duplicate an existing record

For action 1 to 4, you donâ€™t have to select any records, for action 5, you need to select the record you want to duplicate.

Action 3 and 4 will redirect you to a new page which has a dropdown to select the scraper you want to check or uncheck. Action 5 has just been added for makign things easier. There are many fields in this table and often you need to repeat the same information many times, so it seemed smarter to automate this.

In the first 2 cases, you don't need to select records, in the last 2 cases you do need to select the records. When you use the actions or update the records manually, the field show_scraper of the table "scraper variables" (which is still the one used for defining whether a scraper-outputvariable will be scraped or not) will automatically be updated too. The scraper-variable-table is more detailed than the Scraper-scripts-table. The correct order to update is to update the Scraper scripts and then the Scraper variables
