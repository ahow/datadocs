To be able to monitor the normal run, we introduced a logging system that shows, what the server is doing, if there are errors or warnings, how long the tasks have taken, etc. The progress bar is just indicative. Some tasks take seconds, others take hours.

When you click on the button to start the normal run, the scraping starts. After the scraping, the datapoint calculations starts. Then the datapoints from the inputfile are added. Then the metrics are computed. Finally the table will be sorted and exported to excel.

It is not because the message: start scraping glassdoor is shown in the interface, that glassdoor is actually scraped. The website will only be scraped if the settings in the table scraper_output are selected.
