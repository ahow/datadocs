Adding a new scraper (7)

  • 3.1. Insert records into the Scraper variables -table

    • In order to add a new scraper, you need to do the following:

    • Insert records into the Scraper variables-table to configure the output-variables
    • Go to admin-view again (click on base in breadcrumbs)
    • Click Scraper variables
    • Click Add record
    • Give a good name (my scraper-names all start with SCR_O_)
    • Select the CSR-scraper from the list. It should be visible now in the dropdown-menu
    • Select a good abbreviation name. If the scraper is an abbreviation, I use that (GRI/CSR), else i abbreviate it to two letters (FA for facebook, TW for twitter).
    • Copy-paste or type the classname you defined in step 1 in field classname
    • Copy-paste or type the functionname you defined in step 1 in field function
    • Copy-paste or type the outputname of the funtion (name after “return”-field) you defined in step 1 in field function outputfield
    • If the output is a list (as is the case with CSR), you fill in the index in that list in the field function output index.
    • If the output is a dictionnary, you leave the field function output index empty
    • Input 1 should be exactly like in step 2 (name starting with SCR_I)
    • Show_Scraper is by default set to True. I would leave it like that, unless you don’t want the website to be scraped
    • Datatype of the output should be set still (values: TEXT, INTEGER, DECIMAL are accepted)
    • If needed, define an output format, thousand separator and decimal separator for decimal datatypes
    • Click on save
    • Once this record has been added, the scraper will be called by the main program.
    • The variable defined in step 3 (starting with SCR_O_) can be used by the metrics (fields arg1, arg2 and arg3 in Match uploaded columns to scraper inputs in admin view) or can be outputted by itself.
