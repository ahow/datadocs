Empty the database

Normally, you start with an “empty” database, which means there are no companies and most of the tables are empty.

These tables should NOT be emptied (or the program will not function properly anymore):
Scraper scripts: table containing the scripts
Scraper variables: table contains the parameters for the different scrapers
Wikirate metric list: the metric lists for the wikirate-scraper
Match uploaded columns to scraper inputs: match between input-excelfile and python names
Invest IQ Input: can be emptied, but then the investIQ will not work properly
Ranking the brands rankinglists: used for RTB, list of rankinglists
Ranking the brands currency converter table: used for RTB: currency converter table
Define datapoints: datapoint definition Define metrics: define metrics
List of scrapers: list of scrapers used for getting the URLs for the inputfile
Words to exclude from company matching: words that are replaced by an empty string
Scripts to extract all company names: these are other scraper scripts
WIPO Codes and their descriptions: this table can be deleted, but then other parts will not work anymore
Config: constants

The rest of the tables can be emptied safely. Just be aware that in some cases foreign keys have been defined, so if you delete the primary keys, the child records will also be deleted at the same time (you will get a screen specifying what will be deleted and you will need to confirm it). Eg company is used as a foreign key in many tables (eg Company inputs). So when you delete the companies, you will be asked also to delete the company input-records linked to those companies.
