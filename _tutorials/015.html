Input: Import from Database Tables

Scraper variables

Different tables are used as input for python (and can be modified through the admin views):

Scraper variables table contains the information for calling the scraper and generating the output-variables. This table only needs to be modified if a scraper is created or if a scraper is modified. This table is technical, but needs to be set correctly, if you want the scraper to be called and the output of the scraper exported.

In this table, some technical fields need to be set (classname and functionname): they should correspond to the classname and functionnames defined in the table. You also need to define the name of the outputfield and the order in which the output is generated (if the output is a list/array).

At this moment, two different output types are possible in a scraper:

  • Dictionnaries (syntax: starting with “{“ and ending with “}”)
  • Lists (arrays) (syntax: starting with “[“ and ending with “]”)

If the output is a list, you need to define the function output index, which corresponds to the position of the variable in the list.
