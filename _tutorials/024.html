Create input file: Edit outputfile

To create the inputfiles: Step 3 Edit the outputfile

After the fuzzy matching, you will be redirected to the output of this URL http://scrapecalctool.scibizinformatics.com/admin/base/company_source_scraper/

Right now, normally only these columns are filled:

  • Comp name from db (which is the same company name as the company name from the Companies-table)
  • Comp fuzzy name: company name of the best matching company, using fuzzy logic. If it is empty, this means that none of the companies had a higher score than the fuzzy-threshold.
  • Fuzzy score: score of the fuzzy comparison

The rest of the columns remain empty.

The process of fuzzy matching happens as follows:

  • First the total fuzzy score is checked for all the companies
  • If none of the companies have a score higher than the threshold, the partial fuzzy score is computed.
  • If after the comparison, still none of the companies have a score higher than the threshold, the fields remain empty.

As you can see from the results, fuzzy matching does not always work well. Sometimes companies are matched which should not be matched. So manual editing is needed.

Action (used to automate this process a bit):

  • Clear the fuzzy name: select the records and start action: this will empty the two fields (Comp fuzzy name and Fuzzy score) for the selected records
  • Get the best match ... : the opposite of the previous action: 2 fields are filled in
  • Fill the URLs: fills the fields comp best match url (this info comes from table List of URLs from scrapers for fuzzy matching, if the field comp fuzzy name is filled in, this will always be filled) and comp db url (this comes from table Company URLs, if there is a record for the url in the table, else it remains empty)
  • Over-ride the db-url with the best_match_url: only if the best_match_url is filled in
  • Over-ride the db-url with the best_match_url: only if the best_match_url is filled in. The difference with the previous is that the Company URLs is also updated in this case.

In general, after the scraping and fuzzy matching, you run these 2 actions:

  • Fill the URLs: to fill best_match_url and if needed the two columns left of this one
  • Over-ride the db-url with the best_match_url (second option): this creates/updates the records in the Company URLs
